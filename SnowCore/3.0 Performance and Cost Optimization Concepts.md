Resource monitors in Snowflake operate at both account and warehouse levels **independently**.

Setting USE_CACHED_RESULT to FALSE disables the query result cache for the current session. This means Snowflake will not use cached results and will re-execute queries even if identical queries were recently run, ensuring that results are freshly computed.

Snowflake has three types of cache.
The **metadata** cache that lives in the cloud services layer.
The **data cache/loca**l disk cache that lives on the SSD drives in the virtual warehouses, and
The **query result** cache. If a result is small, it will be stored in the cloud services layer, but larger results are going to be stored in the storage layer.
